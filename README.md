# Emperor

> **Work in Progress (Pre-release)** — This project is under active development and has not reached v1.0. Features, APIs, and configurations may change.
>
> ⚠ **Known issues on `main`:** current `main` contains changes that trigger runtime/type errors. Fix scheduled for the next update.

## Vision

Emperor is working toward a **test-time training model** built from a growing cluster of interconnected neural modules, each powered by a sparse universal transformer with persistent hidden state. Signals route between neurons via learned routers, with adaptive computation controlling depth at both the inter-neuron and intra-neuron level. The model does not stop learning after initial training — the cluster grows structurally at inference time, with new edge neurons trained directly on incoming data and frozen core neurons adapted slowly through periodic sparse updates.

The model is assembled entirely from Emperor's **hierarchical configuration framework**, the same system that powers every reference architecture in the project. Every component — attention, feed-forward, routing, halting, normalization — is a swappable option.

See [`VISION.md`](VISION.md) for the full design.

## Quick Start

```bash
# 1. Set up the environment
source env.sh

# 2. Run a simple experiment with a custom log folder
source run_experiment.sh linear --option PRESET --logdir quickstart

# 3. Or run a random hyperparameter search with 10 samples per dataset
source run_experiment.sh linear --option CONFIG --random-search 10 --logdir quickstart

# 4. View training metrics
tensorboard --logdir logs/quickstart
```

Every run iterates over the datasets listed in the model's `DATASET_OPTIONS` (see `models/linear/config.py`), so `--random-search 10` produces `10 × len(DATASET_OPTIONS)` total runs.

## Environment Setup

**Requirements:** Python 3.13 (pinned in `mise.toml`). The package itself supports Python 3.11–3.13.

**Note:** Windows users must use [WSL](https://learn.microsoft.com/en-us/windows/wsl/install) to run the setup scripts.

### Using env.sh (recommended)

```bash
source env.sh
```

On first run, `env.sh` installs `mise` (if missing), uses it to provision the Python version pinned in `mise.toml`, creates a `./torchenv` virtualenv, installs the project from `pyproject.toml`, and activates the environment. On subsequent runs it just re-activates. The script is short and readable — open [`env.sh`](env.sh) if you want to see the exact sequence or adapt it.

## Running Experiments

Use `run_experiment.sh` to train a model.

### List available experiments

```bash
source run_experiment.sh --list
```

```
Available experiments:
  bert
  experts
  linear
  linear_adaptive
  parametric_generator
  parametric_matrix
  parametric_vector
  vit
```

To see the `--option` values accepted by a specific model, pass `--list-options`:

```bash
source run_experiment.sh linear --list-options
```

```
Available options for linear:
  PRESET
  CONFIG
```

Every model exposes two baseline options:

- `PRESET` — runs a single configuration using the default values defined in `config.py`
- `CONFIG` — exploration harness driven by the `SEARCH_SPACE_*` variables in `config.py`; pair with `--grid-search` or `--random-search N` for a full or sampled sweep (see sections below)

### Run a configuration

```bash
source run_experiment.sh linear --option PRESET
source run_experiment.sh vit --option ADAPTIVE
```

`--option` selects which experiment option to run. Each model exposes its own set of options (see `--list-options`). Use `--option` or `--all-options` — one is required.

### Reference Architectures

- **linear** — Fully-connected classifier with configurable depth, activation, and dropout
- **linear_adaptive** — Fully-connected classifier where a generator network produces auxiliary parameters (diagonal weights, bias, memory) that are combined with the default layer parameters at each layer
- **experts** — Mixture of Experts architecture with learned routing between specialized sub-networks
- **vit** — Vision Transformer that splits images into patches and applies transformer encoder blocks
- **parametric_generator** — Adaptive parameter layer where expert weights are generated by a generator network
- **parametric_matrix** — Adaptive parameter layer where expert weights are mixed as full matrices via a shared router
- **parametric_vector** — Adaptive parameter layer where expert weights are mixed as vectors via independent routers with configurable layer stack options
- **bert** — BERT-style language model with transformer encoder, trained on text datasets (PennTreebank, WikiText2)

> [!WARNING]
> The following reference architectures are currently broken on `main` and will fail to run until the next update: `linear_adaptive`, `experts`, `vit`, `parametric_generator`, `parametric_matrix`, `parametric_vector`, `bert`. Only `linear` is known to work end-to-end right now.

Some models expose extra ablation options — run `--list-options` to discover them.

### Datasets

Each run trains one checkpoint per entry in the model's `DATASET_OPTIONS` list (in `config.py`), sequentially — so a single invocation produces `len(DATASET_OPTIONS)` checkpoints, and a search flag multiplies that count. Add or remove dataset classes to change the targets; the full catalog lives under `emperor/datasets/`:

```python
DATASET_OPTIONS: list = [Mnist, FashionMNIST, Cifar10, Cifar100]
```

### Run all configurations

```bash
source run_experiment.sh linear_adaptive --all-options
```

Runs every option in `ExperimentOptions` sequentially across all datasets.

### Run grid search

```bash
source run_experiment.sh linear --option CONFIG --grid-search
source run_experiment.sh linear_adaptive --all-options --grid-search
```

Exhaustively runs every combination in the search space across all datasets.

The search space is defined in each model's `config.py` as variables prefixed with `SEARCH_SPACE_`:

```python
# models/linear/config.py
SEARCH_SPACE_LEARNING_RATE = [1e-4, 1e-3, 1e-2]
SEARCH_SPACE_HIDDEN_DIM = [64, 128, 256]
SEARCH_SPACE_STACK_ACTIVATION = [RELU, SILU, GELU, MISH]
...
```

### Run random search

```bash
source run_experiment.sh linear --option CONFIG --random-search 10
source run_experiment.sh linear_adaptive --all-options --random-search 10
```

Randomly samples 10 configurations from the search space. `--grid-search` and `--random-search` are mutually exclusive and can be combined with either `--option` or `--all-options`.

### Custom log folder

```bash
source run_experiment.sh linear --logdir comparison_run
source run_experiment.sh vit --logdir comparison_run
```

Use the same `--logdir` across different models to group their TensorBoard logs together for comparison.

### Viewing results

Training metrics are logged to `logs/`. Launch TensorBoard to visualize them:

```bash
tensorboard --logdir logs/
```

### Search results

When running `--grid-search` or `--random-search`, two additional files are written:

- **`result.json`** — saved inside each run's log directory with the exact parameters and final metrics for that run
- **`best_results.json`** — saved at `logs/{model}/best_results.json`, updated after every run that makes the top 5

`best_results.json` tracks the top 5 configurations per dataset ranked by `val_accuracy`:

```json
{
  "Mnist": [
    {"rank": 1, "option": "CONFIG", "params": {"learning_rate": "0.001", ...}, "metrics": {"val_accuracy": 0.984, "val_loss": 0.051, ...}},
    {"rank": 2, ...},
    ...
  ],
  "Cifar10": [...]
}
```

The file is updated in place as the search runs — if the process is interrupted, the top 5 found so far is preserved.
