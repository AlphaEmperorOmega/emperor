## First iteration step by step

- create `zero tensors` for
  - `log_never_halt`, `acc_expect_depth` of shape `(sequenceLength, batchSize)`
  - `acc_h` of shape `(sequenceLength, batchSize, embeddingDim)`
  - `index` number representing the current iteration will be used to scale
    the `acc_expect_depth` if further iterations
- store in initial tensors in a tuple `currentAdaptiveComputationState`:
  - `index`
  - `log_never_halt`
  - `acc_h`
  - `acc_expect_depth`
- compute the forward pass sore it into `layerOutputs` with the default
  `paddingMask` or `p_never_halt`
- when using `AdaptiveComputationTimeWrapper` self attention is performed
  so the layer output will be stored in `selfAttentionInput` to be
  used in the next iteration
- initialize the `act_loss` or `adaptiveComputationLoss` to `0`

The output is a tuple that contains

- `currentAdaptiveComputationState`:
- `modelOutput`
- `selfAttentionInput`
- `act_loss`

## Second iteration step by step

- from the previous step the `currentAdaptiveComputationState` is split
  in different variables
  - `index`
  - `log_never_halt`
  - `acc_h`
  - `acc_expect_depth`
- method `_computeGatingLogits` is used to compute for scores in two steps
  - use a neural network to compute scores for each token in the batch
    of shape `(sequenceLength, batchSize, 2)`
  - `log_softmax` is applied on the output of the neural network
- method `_updateHalting` uses the scores generated by `_computeGatingLogits`
  `gatingLogits` and `log_never_halt` and computes the following steps
  - `log_never_halt` is reshaped into `(sequenceLength, batchSize, 1)`
    and added elementwise to scores `gatingLogits` of shape `(sequenceLength, batchSize, 2)`
  - the scores generated in the step above are split into two
    tensors of shape `(sequenceLength, batchSize, 1)` reshaped into
    `(sequenceLength, batchSize)`
    - `p`: not sure what this is used
    - `log_never_halt` this is an updated version of the input
- `acc_h` of shape `(sequenceLength, batchSize, embeddingDim)` is updated by
  reshaping `p` into `(sequenceLength, batchSize, 1)` and by the `output 
of the previous layer` meaning that token in the batch get's a
  score and is amplified or suppressed
- `acc_expect_depth` accumulates `p` scores multiplied by the `index`
  for every single layer, meaning that for the first two layers of
  computation no accumulation accumulation is made
- in `p_never_halt` you exponentiate `log_never_halt` of
  shape `(sequenceLength, batchSize)`
- a mask is created for every element in `p_never_halt` where
  every element smaller that 0.01 is turned off or set to 0
  this will be used as `haltingMask` in the encoder. Important to
  understand that the halting mask is used to mask `softmax` score
  when an expert is chosen in `MixtureOfAttentionHeads` or `MixtureOfExperts`
  where the `p_never_halt` is converted int a tensor of booleans
  based on `zeros` tensor.
