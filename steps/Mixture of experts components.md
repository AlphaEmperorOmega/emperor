# Mixture of experts components

- `computeGating`: computes `topk` probabilities and indexes to sort the input batch tokens assigned to each expert based on the order of the experts in `` [[Parallel experts class]]
- `forward`: input batch is processed by two [[Parallel experts class]] layers separated by an `activation function` and a mixture is performed on the output of the experts by summing the `topk` tokens generated by experts of shape `[batchSize, sequenceLength, topK, outputDim]` input a tensor `[batchSize, sequenceLength, outputDim]`

## 5. TEST MODELS

### 5.1. Single layer of `Mixture Of Experts` on `FashionMNIST`

```{python}
def testMixtureOfExpertsSingleLayer():
  from Emperor.base.utils import Trainer
  from Emperor.base.datasets import FashionMNIST
  from Emperor.experiments import MixtureOfExpertsSingleLayerModel
  from Emperor.config import ModelConfig, ParameterGeneratorOptions

  cfg = ModelConfig(
    inputDim=784,
    hiddenDim=64,
    outputDim=10,
    parameterGeneartorType=ParameterGeneratorOptions.matrix_choice_mixture
  )

  data = FashionMNIST(
    batch_size=cfg.batchSize,
    testDatasetFalg=True,
    testDatasetNumSamples=64
  )

  model = MixtureOfExpertsSingleLayerModel(learningRate=0.1, cfg=cfg)

  trainer = Trainer(max_epochs=30)
  trainer.fit(model, data, printLossFlag=True)

testMixtureOfExpertsSingleLayer()
```
